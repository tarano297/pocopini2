تنظیمات امنیتی که پیاده‌سازی شد:

1. تغییر URL پنل ادمین
   - به جای /admin/ از URL سفارشی استفاده می‌شود
   - در .env تنظیم کنید: ADMIN_URL=secure-panel-2024/

2. IP Whitelist برای پنل ادمین
   - فقط IP های مشخص به ادمین دسترسی دارند
   - در .env تنظیم کنید: ADMIN_IP_WHITELIST=192.168.1.100,203.0.113.5

3. اطلاع‌رسانی تلاش‌های ناموفق
   - ایمیل هشدار به ادمین‌ها ارسال می‌شود
   - ADMIN_LOGIN_ATTEMPT_NOTIFICATION=True

4. Two-Factor Authentication (2FA)
   - احراز هویت دو مرحله‌ای فعال شد
   - TWO_FACTOR_ENABLED=True
   - TWO_FACTOR_FORCE_ADMIN=True

5. محافظت در برابر Brute Force
   - بعد از 5 تلاش ناموفق، حساب قفل می‌شود
   - مدت قفل: 1 ساعت

6. Rate Limiting پیشرفته
   - محدودیت درخواست برای هر endpoint

7. رمزنگاری قوی
   - استفاده از Argon2 برای هش رمز
   - حداقل 10 کاراکتر

مراحل راه‌اندازی Production:

1. کپی کردن فایل تنظیمات:
   cp .env.production.example .env.production

2. ویرایش .env.production و تنظیم مقادیر واقعی

3. تغییر ADMIN_URL به یک مقدار تصادفی و پیچیده

4. اضافه کردن IP های مجاز به ADMIN_IP_WHITELIST

5. تنظیم ایمیل برای دریافت هشدارها

6. فعال‌سازی HTTPS و SSL

7. استفاده از PostgreSQL به جای SQLite

8. فعال‌سازی Redis برای Cache

نکات امنیتی مهم:

- هرگز DEBUG=True در Production نگذارید
- SECRET_KEY را تغییر دهید و محرمانه نگه دارید
- از رمزهای قوی برای دیتابیس استفاده کنید
- HTTPS را حتماً فعال کنید
- لاگ‌های امنیتی را مرتب بررسی کنید
- بک‌آپ منظم از دیتابیس بگیرید
