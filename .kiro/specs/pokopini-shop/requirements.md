# مستندات الزامات - فروشگاه آنلاین پوکوپینی

## مقدمه

پوکوپینی یک فروشگاه آنلاین حرفه‌ای برای فروش لباس کودکان است که با استفاده از Django REST Framework در بک‌اند و React در فرانت‌اند پیاده‌سازی می‌شود. این سیستم امکان مدیریت محصولات در سه دسته اصلی (نوزاد، دخترانه، پسرانه) و چهار فصل (زمستان، بهار، تابستان، پاییز) را فراهم می‌کند. کاربران می‌توانند محصولات را مشاهده، خریداری و نظر ثبت کنند.

## واژه‌نامه

- **سیستم پوکوپینی**: پلتفرم فروشگاه آنلاین که شامل بک‌اند Django و فرانت‌اند React است
- **کاربر**: شخصی که در سیستم ثبت‌نام کرده و می‌تواند خرید انجام دهد
- **محصول**: لباس کودک که دارای ویژگی‌های نام، توضیح، قیمت، دسته، فصل، عکس، رنگ و سایز است
- **سفارش**: مجموعه محصولاتی که کاربر خریداری کرده است
- **نظر**: بازخورد کاربر درباره محصول پس از خرید
- **دسته محصول**: یکی از سه گروه نوزاد، دخترانه یا پسرانه
- **فصل**: یکی از چهار فصل زمستان، بهار، تابستان یا پاییز
- **سبد خرید**: لیست موقت محصولاتی که کاربر قصد خرید آنها را دارد
- **API**: رابط برنامه‌نویسی که ارتباط بین فرانت‌اند و بک‌اند را برقرار می‌کند
- **پنل ادمین**: رابط مدیریتی Django برای مدیران سیستم
- **تاریخ شمسی**: تقویم هجری شمسی (جلالی) مورد استفاده در ایران
- **تومان**: واحد پولی ایران

## الزامات

### الزام ۱: مدیریت کاربران

**User Story:** به عنوان یک کاربر، می‌خواهم بتوانم در سیستم ثبت‌نام کنم و وارد شوم تا بتوانم خرید انجام دهم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک API برای ثبت‌نام کاربر با فیلدهای نام، ایمیل، یوزرنیم و پسورد فراهم کند
2. سیستم پوکوپینی باید یک API برای ورود کاربر با یوزرنیم و پسورد فراهم کند
3. هنگامی که کاربر با موفقیت وارد شود، سیستم پوکوپینی باید یک توکن احراز هویت JWT صادر کند
4. سیستم پوکوپینی باید پسورد کاربران را به صورت هش‌شده در پایگاه داده ذخیره کند
5. سیستم پوکوپینی باید اعتبارسنجی ایمیل و یوزرنیم یکتا را انجام دهد

### الزام ۲: مدیریت آدرس‌های کاربر

**User Story:** به عنوان یک کاربر، می‌خواهم بتوانم آدرس‌های خود را مدیریت کنم تا سفارشات به آدرس صحیح ارسال شوند

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک API برای افزودن آدرس جدید برای کاربر احراز هویت شده فراهم کند
2. سیستم پوکوپینی باید یک API برای نمایش لیست آدرس‌های کاربر احراز هویت شده فراهم کند
3. سیستم پوکوپینی باید یک API برای ویرایش آدرس موجود کاربر احراز هویت شده فراهم کند
4. سیستم پوکوپینی باید یک API برای حذف آدرس کاربر احراز هویت شده فراهم کند
5. سیستم پوکوپینی باید امکان تعیین یک آدرس به عنوان آدرس پیش‌فرض را فراهم کند

### الزام ۳: مدیریت محصولات

**User Story:** به عنوان یک مدیر، می‌خواهم بتوانم محصولات را مدیریت کنم تا کاتالوگ فروشگاه به‌روز باشد

#### معیارهای پذیرش

1. سیستم پوکوپینی باید مدل محصول با فیلدهای نام، توضیح، قیمت، دسته، فصل، عکس، رنگ و سایز داشته باشد
2. سیستم پوکوپینی باید دسته محصول را به یکی از سه مقدار نوزاد، دخترانه یا پسرانه محدود کند
3. سیستم پوکوپینی باید فصل محصول را به یکی از چهار مقدار زمستان، بهار، تابستان یا پاییز محدود کند
4. سیستم پوکوپینی باید از طریق پنل ادمین Django امکان افزودن، ویرایش و حذف محصولات را فراهم کند
5. سیستم پوکوپینی باید قیمت محصولات را به تومان ذخیره و نمایش دهد

### الزام ۴: نمایش و فیلتر محصولات

**User Story:** به عنوان یک کاربر، می‌خواهم بتوانم محصولات را مشاهده و فیلتر کنم تا محصول مورد نظرم را پیدا کنم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک API برای نمایش لیست تمام محصولات فراهم کند
2. سیستم پوکوپینی باید امکان فیلتر محصولات بر اساس دسته را فراهم کند
3. سیستم پوکوپینی باید امکان فیلتر محصولات بر اساس فصل را فراهم کند
4. سیستم پوکوپینی باید امکان فیلتر محصولات بر اساس رنگ را فراهم کند
5. سیستم پوکوپینی باید امکان فیلتر محصولات بر اساس سایز را فراهم کند
6. سیستم پوکوپینی باید یک API برای نمایش جزئیات یک محصول خاص فراهم کند

### الزام ۵: مدیریت سبد خرید و سفارش

**User Story:** به عنوان یک کاربر، می‌خواهم بتوانم محصولات را به سبد خرید اضافه کنم و سفارش ثبت کنم تا خرید خود را تکمیل کنم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک API برای افزودن محصول به سبد خرید کاربر احراز هویت شده فراهم کند
2. سیستم پوکوپینی باید یک API برای نمایش محتویات سبد خرید کاربر احراز هویت شده فراهم کند
3. سیستم پوکوپینی باید یک API برای حذف محصول از سبد خرید کاربر احراز هویت شده فراهم کند
4. سیستم پوکوپینی باید یک API برای ثبت سفارش از سبد خرید کاربر احراز هویت شده فراهم کند
5. هنگامی که سفارش ثبت شود، سیستم پوکوپینی باید تاریخ سفارش را به صورت شمسی ذخیره کند
6. سیستم پوکوپینی باید وضعیت سفارش را با مقادیر در انتظار، در حال پردازش، ارسال شده و تحویل داده شده مدیریت کند

### الزام ۶: مدیریت نظرات

**User Story:** به عنوان یک کاربر، می‌خواهم بتوانم برای محصولاتی که خریده‌ام نظر ثبت کنم تا تجربه خود را با دیگران به اشتراک بگذارم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک API برای ثبت نظر برای محصول توسط کاربر احراز هویت شده فراهم کند
2. اگر کاربر محصول را خریداری نکرده باشد، سیستم پوکوپینی باید ثبت نظر را رد کند
3. سیستم پوکوپینی باید نظرات را با کاربر و محصول مرتبط ذخیره کند
4. سیستم پوکوپینی باید یک API برای نمایش نظرات یک محصول خاص فراهم کند
5. سیستم پوکوپینی باید امکان ویرایش و حذف نظر توسط کاربر سازنده نظر را فراهم کند

### الزام ۷: پنل مدیریت

**User Story:** به عنوان یک مدیر، می‌خواهم از پنل ادمین Django استفاده کنم تا تمام جنبه‌های فروشگاه را مدیریت کنم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید پنل ادمین Django را برای مدیریت محصولات فعال کند
2. سیستم پوکوپینی باید پنل ادمین Django را برای مدیریت کاربران فعال کند
3. سیستم پوکوپینی باید پنل ادمین Django را برای مدیریت سفارش‌ها فعال کند
4. سیستم پوکوپینی باید پنل ادمین Django را برای مدیریت نظرات فعال کند
5. سیستم پوکوپینی باید رابط پنل ادمین را به زبان فارسی نمایش دهد

### الزام ۸: رابط کاربری فرانت‌اند

**User Story:** به عنوان یک کاربر، می‌خواهم یک رابط کاربری زیبا و کاربردی داشته باشم تا تجربه خرید راحتی داشته باشم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید یک صفحه اصلی با نمایش دسته‌بندی‌ها و محصولات جدید داشته باشد
2. سیستم پوکوپینی باید یک صفحه جزئیات محصول با عکس، توضیح، قیمت و نظرات داشته باشد
3. سیستم پوکوپینی باید صفحات ثبت‌نام و ورود کاربر داشته باشد
4. سیستم پوکوپینی باید یک صفحه مدیریت آدرس‌های کاربر داشته باشد
5. سیستم پوکوپینی باید یک صفحه سبد خرید و ثبت سفارش داشته باشد
6. سیستم پوکوپینی باید از کتابخانه Tailwind CSS برای استایل استفاده کند
7. سیستم پوکوپینی باید تمام محتوا و رابط کاربری را به زبان فارسی نمایش دهد
8. سیستم پوکوپینی باید طراحی واکنش‌گرا برای موبایل، تبلت و دسکتاپ داشته باشد

### الزام ۹: بومی‌سازی ایرانی

**User Story:** به عنوان یک کاربر ایرانی، می‌خواهم سیستم با فرهنگ و زبان من سازگار باشد تا راحت‌تر استفاده کنم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید تمام تاریخ‌ها را به صورت شمسی (جلالی) نمایش دهد
2. سیستم پوکوپینی باید واحد پولی را به تومان نمایش دهد
3. سیستم پوکوپینی باید جهت متن را از راست به چپ (RTL) تنظیم کند
4. سیستم پوکوپینی باید از فونت فارسی مناسب استفاده کند
5. سیستم پوکوپینی باید اعداد را به صورت فارسی نمایش دهد

### الزام ۱۰: امنیت و احراز هویت

**User Story:** به عنوان یک کاربر، می‌خواهم اطلاعات من امن باشد و فقط من به حساب کاربری‌ام دسترسی داشته باشم

#### معیارهای پذیرش

1. سیستم پوکوپینی باید از JWT برای احراز هویت API استفاده کند
2. سیستم پوکوپینی باید APIهای محافظت شده را فقط برای کاربران احراز هویت شده در دسترس قرار دهد
3. سیستم پوکوپینی باید از HTTPS برای ارتباطات امن استفاده کند
4. سیستم پوکوپینی باید از CORS برای کنترل دسترسی فرانت‌اند استفاده کند
5. سیستم پوکوپینی باید اعتبارسنجی ورودی را در سمت سرور انجام دهد
